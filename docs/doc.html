LITE (LoopIndex Tracking Enhancement) for CKEditor

Installation

Drop the two folders, common and lite, into the ckeditor plugins folder

Configuration

  1. Include the lite plugin's interface in your html, e.g. 
    	<script type="text/javascript" src="ckeditor/plugins/lite/lite_interface.js"></script>
  2. Verify that the editor's configuation contains a lite section by adding the following lines to ckeditor's config.js: 
    	var lite = config.lite|| {};
	config.lite = lite;
  3. Add the lite plugin to ckeditor. The simplest way to do this is by adding the following line to ckeditor's config.js:
    	config.extraPlugins = 'lite';
    If your configuration already contains some plugin tweaking, just make sure that lite is included in the editor's plugins list.
  4. If your page doesn't include jQuery, the lite plugin will load it if you provide it with the relative path to the jquery source. If you are going to use the copy distributed with the plugin, add the following line to config.js: 
    	config.lite.jQueryPath = "../common/jquery.min.js";
  5. By default, lite loads an uncompressed version of ice, located in its js folder. To load ice from a different location, or load modified scripts, edit config.js and set a value to config.lite.includes, e.g. 
    	config.lite.includes = ["js/ice-all-sources.min.js", "js/plugins/ice-plugins.min.js"];
  6. Select the lite commands you want to include in ckeditor's toolbar. By default all the commands in lite_interace.js are available in the toolbar. To configure this, edit config.js and set a value to the config.lite.commands, e.g. 
    // display only the show/hide, accept all and reject all commands.
	config.lite.commands = [LITE.Commands.TOGGLE_SHOW, LITE.Commands.ACCEPT_ALL, LITE.Commands.REJECT_ALL];
  7. You may also tweak the configuration in your code by listening to the CKEditor configLoaded event and then modifying the editor's config.lite object.

Methods

  • Get a reference to the lite plugin

    To interact with the lite plugin, either access it through the editor's plugins.lite property, or listen to the event LITE.Events.INIT fired by the editor instance. The data member of the event will contain a property called lite which references the lite plugin instance, initialized and ready for action.

  • toggleTracking(bTrack, bNotify)

    Toggles change tracking in the editor. If bTrack is undefined, change tracking is toggled, otherwise it is set to the boolean value of bToggle. if bNotify is false, the LITE.Events.TRACKING event is not fired.

  • toggleShow(bTrack, bNotify)

    Change the visibility (special style) of tracked changes. If bTrack is undefined, the visibilty is toggled, otherwise it is set to the boolean value of bTrack. if bNotify is false, the LITE.Events.SHOW_HIDE event is not fired.

  • acceptAll

    Accept all the pending changes.

  • rejectAll

    Reject all the pending changes.

  • setUserInfo(info)

    Sets the name and id of the current user. Each tracked change is associated with a user id. The info parameter contains two members - id and name
